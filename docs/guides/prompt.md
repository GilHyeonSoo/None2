# UAM Network 논문 작성 프로젝트 — Antigravity Agent 지침서

## 프로젝트 개요

본 프로젝트는 **학부생 신분**으로 작성하는 **KCI(한국학술지인용색인) 등재지 기준** 학술 논문입니다.
논문의 핵심 주제는 **UAM(Urban Air Mobility) 환경에서의 네트워크 연속성 및 고속 연결 유지 방안**입니다.
공중 이동 차량(eVTOL, UAV 등)이 이동 중 네트워크가 끊기지 않고 빠르게 연결을 유지하는 방법을 연구합니다.

---

## ⚠️ Agent 필수 준수 사항 (모든 작업에 공통 적용)

- 모든 문장은 **학술 논문 어조**로 작성한다 (구어체, 줄임말, 감탄사 절대 금지).
- 글쓰기 참조 예시 파일인 `@논문/docs/references/examples/manuscript1.txt`, `@논문/docs/references/examples/manuscript2.txt`를 **반드시** 먼저 읽고, 해당 문체·어조·구조·표현 방식을 그대로 따른다. 토씨 하나까지 참조 예시 스타일을 유지한다.
- KCI 기준에 따라 **국문 초록(200~300자)**, **영문 초록(Abstract)**, **키워드 3~5개(국문·영문)** 를 논문 서두에 반드시 포함한다.
- 참고문헌은 투고 학술지의 양식(APA 또는 Chicago)을 따르며, DOI가 존재할 경우 반드시 포함한다.
- 논문 전체 구성 순서: **제목 → 초록 → 키워드 → 서론 → 관련 연구 → 시나리오 → 성능 평가 → 결론 → 참고문헌**
- Agent는 어떤 섹션을 작성하기 전에도 해당 섹션과 관련된 **참조 파일을 먼저 읽고** 작성을 시작한다.

│   │       ├── [국내논문2].pdf        ← 관련 연구 국내 논문 2
│   │       ├── [해외논문1].pdf        ← 관련 연구 해외 논문 1

## 📝 섹션별 작성 지침

### 1. 서론 (Introduction)

**구조:** 대분류 제목 하위에 소분류 소제목을 두어 계층적으로 구성한다.

**분량:** 총 3~4개 문단, 문단당 6~7문장으로 구성한다.

**각 문단별 내용 지침:**

| 문단 | 내용 |
|------|------|
| 1문단 | UAM의 등장 배경과 사회적 필요성, 도심 항공 모빌리티 시장의 성장 |
| 2문단 | UAM 운용 시 네트워크 연결 유지의 중요성, 기존 지상 셀룰러망의 한계 |
| 3문단 | 기존 연구의 공백(research gap) 제시, 본 논문의 연구 목적 및 기여점 |
| 4문단 (선택) | 논문 전체 구성 안내 (각 섹션이 무엇을 다루는지 1~2문장씩 소개) |

**작성 시 유의사항:**
- 연구 배경 → 문제 제기 → 기존 연구 한계 → 본 연구 목적의 흐름을 유지한다.
- 주장마다 참고문헌을 인용하여 논거를 뒷받침한다.
- 독자가 UAM 네트워크 문제의 심각성을 이해하고, 본 연구의 필요성에 공감할 수 있도록 서술한다.

---

### 2. 관련 연구 (Related Works)

**참조 파일:** `@논문/docs/references/related_works/` 폴더 내 PDF 4개 (국내 2개, 해외 2개)

**작성 절차:**

1. Agent는 4개의 PDF 파일을 모두 읽는다.
2. 각 논문에서 다음 정보를 추출한다:
   - 논문 제목, 저자, 발행 연도, 게재 학술지
   - 핵심 연구 문제 및 접근 방법
   - 주요 실험 결과 및 결론
   - 본 논문과의 연관성 및 차별점
3. 국내 논문 2편은 묶어서, 해외 논문 2편은 묶어서 서술하거나, 주제별로 묶어 비교 서술한다.
4. 단순 나열이 아니라 **각 논문의 한계와 본 논문이 이를 어떻게 보완하는지**를 명시한다.
5. 참고문헌 인용 번호는 본문에 `[번호]` 형식으로 표기한다.

---

### 3. 시나리오 (Proposed Scenario / Architecture)
* **목적:** 제안하는 네트워크 연결 유지 기법의 처음 시작부터 끝까지의 전체 프로세스 설명.

* **작성 지침 (매우 중요):** 독자나 심사위원이 어떠한 논리적 반박이나 의문(loophole)도 제기할 수 없도록 **방어적이고 치밀하게 작성**할 것.

* **세부 사항:** * 각 단계를 극도로 세분화하고 구체적으로 설명할 것.
    * 모호한 표현을 배제하고, 특정 상황(예: UAM 기체의 고도 변화, 핸드오버 발생 시점, 신호 간섭 등)에서 시스템이 어떻게 대처하는지 명확한 인과관계로 서술할 것.

> 실제 UAM 네트워크 맥락에 맞게 각 단계를 풍부하고 구체적으로 확장하여 작성한다. 각 단계에서 "왜 이 방식인가"에 대한 이유를 반드시 포함한다.

---

### 4. 성능 평가 (Performance Evaluation)

**⚠️ 핵심 주의사항:** 그래프 이미지 파일을 논문에 직접 삽입하지 않는다.
Agent는 반드시 **원본 데이터(CSV, JSON, 수치 테이블 등)를 직접 읽어**, 수치를 분석하고 논문 텍스트 내에 결과를 서술한다.

**데이터 파일 위치:** `@논문/results/raw/` (실험 완료 후 원본 데이터 저장 경로)

#### 4.1 Reward 설계

Agent는 다음 구조로 Reward 함수를 서술한다:

- **Reward 구성 요소:** 핸드오버 횟수, 네트워크 지연(latency), 처리량(throughput), 간섭(interference), 연결 유지율
- **Reward 수식:** 각 요소의 가중치와 결합 방식을 수식으로 명시
- **Positive/Negative Reward 조건:** 연결 유지 시 양의 보상, 핸드오버 실패·지연 초과 시 음의 보상

#### 4.2 Ablation Study

- 각 Reward 구성 요소를 하나씩 제거하거나 변경했을 때 성능 변화를 비교 분석한다.
- 원본 데이터를 기반으로 조건별 수치 변화를 표(Table)로 정리한다.
- 어떤 요소가 전체 성능에 가장 큰 영향을 미치는지 결론을 도출한다.

#### 4.3 실험 결과 작성 방법 (Agent 필수 준수)

```

[데이터 분석 절차]

1. @논문/results/raw/ 경로의 원본 데이터 파일을 읽는다.
2. 수치 데이터를 직접 파싱하여 평균, 최솟값, 최댓값, 표준편차 등을 계산한다.
3. 비교 실험(baseline vs. proposed)의 수치 차이를 퍼센트 및 절댓값으로 명시한다.
4. 그래프 시각화는 Antigravity 내 코드 실행으로 생성하되,
논문 본문 텍스트에는 그래프 이미지가 아닌 수치 분석 결과를 서술한다.
5. 예: "제안 알고리즘은 기존 대비 핸드오버 횟수를 평균 32.4% 감소시켰으며,
평균 지연은 12.7ms에서 8.3ms로 개선되었다 (Table 2 참조)."
```

#### 4.4 그래프 생성 (코드 작성)

- 원본 데이터를 기반으로 Python(matplotlib / seaborn / plotly) 코드를 작성하여 그래프를 생성한다.
- 그래프 종류: 학습 곡선(Learning Curve), 누적 Reward, Ablation 비교 막대그래프, 지연/처리량 비교
- 생성된 그래프 이미지는 논문의 Figure로 삽입하고 Figure 번호와 캡션을 붙인다.
- **단, 논문 본문 분석 내용은 이미지에 의존하지 않고 수치 텍스트로 서술한다.**

## 🔖 KCI 논문 형식 체크리스트

Agent는 논문 초안 완성 후 아래 항목을 전부 점검한다:

- [ ] 제목: 국문 + 영문, 20단어 이내, 핵심 키워드 포함
- [ ] 국문 초록: 200~300자, 목적·방법·결과·결론 포함
- [ ] 영문 초록(Abstract): 150~250단어
- [ ] 키워드: 국문 3~5개, 영문 3~5개
- [ ] 각주·인용: 본문 내 `[번호]` 형식 일관 적용
- [ ] 참고문헌: APA 또는 Chicago 형식, DOI 포함, 알파벳/가나다 순 정렬
- [ ] 그림(Figure): 번호 + 영문 캡션 포함
- [ ] 표(Table): 번호 + 영문 캡션 포함
- [ ] 수식: 번호 부여, LaTeX 스타일 표기
- [ ] 학부생 신분에 맞는 겸손하고 명확한 학술 어조 유지

## 🚀 Agent 작업 시작 순서

1. `@논문/docs/references/examples/manuscript1.txt` 및 `@논문/docs/references/examples/manuscript2.txt` 읽기 → 문체 학습
2. `@논문/docs/references/related_works/` 내 PDF 4개 읽기 → 관련 연구 내용 파악
3. 서론 → 관련 연구 → 시나리오 → 성능 평가 순서로 초안 작성
4. 성능 평가 단계에서 `@논문/results/raw/` 원본 데이터 파일 읽기 → 수치 분석 → 결과 서술
5. KCI 체크리스트 항목 전체 점검 후 최종본 출력
```
